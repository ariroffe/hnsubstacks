* View all domains in the table:

SELECT * FROM custom_domains;

* View all pending domains:

SELECT * FROM custom_domains
WHERE status = 'pending';

---

* View the oldest 10 pending approval domains:

SELECT * FROM custom_domains
WHERE status = 'pending'
ORDER BY submitted_at ASC
LIMIT 10;

* Newest 10 pending: replace ASC for DESC

---

* Approve a domain:

UPDATE custom_domains
SET status = 'approved',
    validated_at = CURRENT_TIMESTAMP,
    validation_notes = COALESCE(validation_notes, '') || '; manual:approve'
WHERE domain = '<good-example.com>';

* Reject a domain:

UPDATE custom_domains
SET status = 'rejected',
    validated_at = CURRENT_TIMESTAMP,
    validation_notes = COALESCE(validation_notes, '') || '; manual:reject'
WHERE domain = 'google.com';

---

* View first 10 flagged domains:

SELECT id, domain, status, flagged, flag_reviewed, submitted_at 
FROM custom_domains 
WHERE flagged = 1 
ORDER BY submitted_at ASC 
LIMIT 10;

Last 10: replace ASC for DESC

* Review a flag and keep the domain approved:

UPDATE custom_domains 
SET flag_reviewed = 1,
    validated_at = CURRENT_TIMESTAMP
WHERE domain = 'www.noahpinion.blog';

* Review a flag and reject the domain:

UPDATE custom_domains 
SET flag_reviewed = 1,
    status = 'rejected',
    validation_notes = COALESCE(validation_notes, '') || '; flag: rejected',
    validated_at = CURRENT_TIMESTAMP
WHERE domain = '<example.com>';

---

* Delete a domain (we shouldn't have to do this):

DELETE FROM custom_domains
WHERE domain = '<google.com>';

---

To do any of this in the local db (replace the query for the one you want):

npx wrangler d1 execute DB --local --command="SELECT * FROM custom_domains;"

---

To run a migration:

npx wrangler d1 execute DB --remote --file=./migrations/0002_flagging.sql
npx wrangler d1 execute DB --local --file=./migrations/0002_flagging.sql

Backup the entire database (from hnsubstacks/worker):

npm run backup_db
